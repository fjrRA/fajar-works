## Mengapa Membuat Fajar Works?

Awalnya saya memikirkan website portfolio hanya sebagai tempat untuk menampilkan beberapa proyek. Namun, semakin lama saya menyadari bahwa proyek saja belum tentu dapat menjelaskan proses di balik pekerjaan yang saya lakukan.

Karena itu, Fajar Works dirancang sebagai gabungan antara portfolio, catatan teknis, learning log, dan arsip perkembangan pribadi.

## Arah Desain

Saya memilih gaya minimalis industrial dan editorial. Saya tidak ingin tampilannya terlalu menyerupai landing page startup atau dashboard SaaS.

Struktur visualnya dibangun menggunakan garis, grid, tipografi besar, metadata monospace, dan warna oranye sebagai penanda.

## Arsitektur Konten

Konten website disimpan menggunakan JSON dan Markdown. Pendekatan ini memungkinkan website tetap statis tanpa database cloud atau server backend yang harus terus berjalan.

Nantinya, data tersebut dapat dikelola melalui local CMS dan diekspor kembali menjadi file untuk website publik.

## Kondisi Saat Ini

Fondasi global, navigasi, footer, homepage, selected work, capabilities, dan current focus sudah mulai terbentuk.

Website ini masih berkembang, tetapi sekarang sudah memiliki sistem visual dan struktur konten yang lebih jelas daripada sekadar halaman portfolio tunggal.
